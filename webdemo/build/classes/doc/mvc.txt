MVC패턴
1 Model-View-Controller패턴은 스몰토크라는 프로그램 언어에서
  사용되는 패턴이었는데, 스윙과 같은 UI컴포넌트로 그 활용
  범위가 확장되었다.

2 Model : 비즈니스 영역의 상태 정보를 처리한다.
  View : 비지니스 영역에 대한 프레젠테이션 뷰를 담당한다.
  Controller : 사용자의 입력 및 흐름 제어를 담당한다.

3 MVC패턴의 흐름도
  1) 사용자가 서버에 페이지를 요청을 하면 컨트롤러에서 받는다.
  2) 컨트롤러에서는 요청받은 정보를 처리하기 위해서 모델과 
      상호작용을 한다.
  3) 컨트롤러는 응답해줄 뷰페이지를 선택한다.
  4) 선택된 뷰페이지를 클라이언트에 응답한다.



4 MVC패턴과 모델2 구조의 매핑
  Model = 자바
  View = JSP   (script 컴파일 없이 사용 가능)
  Controller = Servlet (웹을 지원하는 자바라고 생각하면 됨 / 컴파일을 해야함) (자바는 로컬 프로그램. 웹을 지원X)
  이게 지금 쓰는 방식. 
  서블릿이 나오고 jsp가 나옴. jsp만 사용하면 보안에 취약. 소스가 그대로 올라가기 때문.
  

 (MVC패턴 모델1 방식
	Model = 자바
	View / Controller = JSP)
